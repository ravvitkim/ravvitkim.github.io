Easy-Quality

지능형 SOP 규정 분석 시스템

1. 기획 배경 (문제 파악)

현업 경험이 풍부한 팀원들과 식약처 가이드라인 프로젝트를 논의하던 중, "규정(SOP) 하나가 바뀔 때 연관된 타 부서 매뉴얼까지 사람이 일일이 찾아 고쳐야 해서 누락과 감사 지적이 빈번하다"는 실무의 짙은 고충을 전해 들었습니다. 이를 기술적으로 해결하고자, 문서 개정 시 보이지 않는 파급 효과(Impact)까지 자동으로 역추적해 주는 지능형 시스템을 기획하게 되었습니다.

2. 프로젝트 개요 (해결책)

SOP(표준작업지침서) 규정 문서를 자동 분석하고 질의응답을 제공하는 AI 에이전트 시스템입니다. 단순 비교 툴이 아닌, 그래프DB를 활용한 관계 추적과 ReAct 기반의 비교 분석을 통해 자동화된 감사 대응 리포트 생성을 주 목표로 합니다.

3. 담당 역할

버전비교에이전트 개발 관계추적에이전트 개발 시스템 아키텍처 설계

4. 핵심 개발 내용 및 활용 기술 (도전 과제)

[1] 버전비교 AI (Comparison Agent)

"스스로 생각하고 필요한 도구를 꺼내 쓰는 똑똑한 문서 분석 AI"

  • 자율적인 문제 해결 구조 (ReAct): 사용자가 질문하면 단순히 고정된 답변을 반환하는 것이 아니라, 문서 이력 조회 → 버전 비교 → 영향 분석 순으로 스스로 계획을 수립하고 실행하는 AI 에이전트를 설계했습니다.
  • 다양한 도구 활용: 버전을 비교하거나, 참고 문서를 찾아보는 등 AI가 사용할 수 있는 여러 '도구'들을 쥐어주어 복잡한 현업의 질문에도 유연하게 대처할 수 있게 만들었습니다.
  • 보기 편한 맞춤형 리포트: 문서에서 "추가된 내용(ADDED)", "삭제된 내용(DELETED)" 등을 색상과 표로 명확하게 깔끔하게 정리하여, 사용자가 한눈에 변경 사항을 파악할 수 있도록 UI를 최적화했습니다.
[2] 관계추적 AI (Graph Agent)

"문자 간 관계를 분석하여 개정 시 영향을 받는 다른 조항들을 추적하는 AI"

  • 사용자 의도(Intent) 자동 추출: 자연어 질문을 LLM이 분석하여 사용자가 원하는 분석 유형(영향 분석, 참조 조회 등)과 대상 문서를 정확히 파악하고, 그에 맞는 처리 경로로 자동 연결합니다.
  • 그래프 DB(Neo4j)를 활용한 입체적 분석: 문서들을 단순히 리스트가 아닌 '지도'처럼 연결(그래프)해 두었습니다. A 문서의 1항을 고치면, 이를 참고하고 있는 B 문서의 3항까지 연쇄적으로 찾아냅니다.
  • 자동 시각화 (한눈에 보는 파급 효과): 이 복잡한 영향 관계를 말로만 설명하는 게 아니라, 다이어그램(시각화 그래프)으로 자동 변환해 주어 누구나 직관적으로 리스크를 확인할 수 있게 했습니다.
[3] 지능형 문서 전처리 시스템 (AI가 글을 잘 읽게 만드는 과정)

"AI가 문서를 사람처럼 '의미 단위'로 정확하게 이해하도록 가르치는 과정"

  • 깨진 문서 복원 (Fallback): 표나 레이아웃이 복잡한 PDF 문서를 AI가 읽게 하면 글자가 뒤죽박죽 섞이는 문제가 있습니다. 이를 방지하기 위해 1차 시도 실패 시 즉각 다른 변환 방식을 적용하는 안전망을 구축했습니다.
  • 반복되는 쓰레기 데이터(노이즈) 제거: 매 페이지마다 반복되는 '페이지 번호'나 '회사 로고' 같은 의미 없는 텍스트를 AI가 자동으로 걸러내어, 검색 품질을 떨어뜨리는 원인을 제거했습니다.
  • 조항 단위(1항, 2항 등)의 똑똑한 분할: 긴 문서를 단순히 길이에 따라 무식하게 자르는 것이 아니라, "제1조 목적", "제2조 적용 범위" 등 법안/규정 특유의 '조항' 단위로 문맥이 유지되게 나누어 AI 통찰력을 극대화했습니다.

5. 성과 및 기여도

  • 고품질 RAG 데이터 파이프라인 구축 (Garbage In, Garbage Out 차단): 다단계 조항 구조의 정규식 파싱과 상위 조항 제목의 강제 병합 적용, LLM을 통한 행위자(Actor)/목적 기반의 다차원 메타데이터화를 통해 벡터 검색의 정확도를 비약적으로 끌어올리는 하이브리드 파이프라인을 고안했습니다.
  • 그래프DB 기반 파급 효과 역추적: 위와 같이 정제된 튼튼한 데이터 파이프라인을 토대로 Neo4j 문서 간 지식 그래프를 구축하였으며, 해당 경로를 역추적하는 에이전트를 개발하여 "보이지 않는 파급 효과"를 즉시 파악할 수 있게 되었습니다.
  • 스마트한 종합 리포트 자동화: 양질의 문서 컨텍스트를 활용한 ReAct 기반 버전비교에이전트를 설계하여, 실무자가 변경 내용을 물으면 핵심 내용과 요주의 문서 리스트를 종합 리포트로 즉각 자동 생성해 냅니다.
  • 업무 효율 향상 및 리스크 통제: 문서 개정에 따른 누락 및 수정 오류를 원천 차단하고, 수동으로 진행해 왔던 업무 영향성 검토 시간을 획기적으로 단축시켰습니다.

스마트폰/웹캠 영상과 사용자의 음성 질문을 결합하여 멀티모달 AI 스택을 통해 실시간으로 상황을 설명해 주는 형태의 시각장애인용 실시간 영상/음성 분석 시스템입니다.

3. 담당 역할

Core AI 파이프라인 최적화 언어 모델 튜닝

4. 주요 도전 과제

  • 부족한 컴퓨터 자원(메모리) 극복: 시각 AI 모델은 눈(카메라)으로 들어온 영상을 수학적으로 처리하기 때문에 방대한 컴퓨터 메모리(VRAM)를 필요로 합니다. 6GB라는 제한된 개인용 PC 환경에서 무거운 AI를 돌리다 보니 멈추거나 튕기는 병목 현상이 발생했습니다.
  • AI의 엉뚱한 대답(환각 현상) 제어: AI가 영상을 분석해 영어로 설명한 뒤, 이를 다시 한국어로 번역하는 과정을 거칩니다. 이 과정에서 여러 언어 모델이 충돌하며 갑자기 중국어를 뱉어내거나 기계어를 섞어 쓰는 심각한 번역 오류(환각)가 발생했습니다.
  • 끊김 없는 실시간 대화 환경 구축: 시각장애인 분들이 길을 걷거나 물건을 집을 때 사용해야 하므로, 영상과 음성 질문이 들어오면 수초 내에 바로 피드백이 나가는 '지연 없는 실시간(Streaming) 환경'을 만드는 것이 핵심이었습니다.

5. 성과 및 기여도

  • 무거운 AI를 가볍고 빠르게 : AI 모델이 계산할 때 쓰는 숫자 데이터의 크기(torch.float16 반정밀도)를 절반으로 줄여 메모리 사용량을 대폭 줄였습니다. 덕분에 6GB 이하의 일반 PC 환경에서도 성능을 잃지 않고 빠르고 안정적으로 AI가 작동하게 만들었습니다.
  • 자연스럽고 정확한 한국어 번역 파이프라인 (중국어 혼입 차단): Qwen2-VL 모델의 영어 설명을 한국어로 매끄럽게 번역하는 시스템을 구축했습니다. 특히 AI에게 "절대 다른 언어를 섞지 마"라는 강력한 규칙(프롬프팅 및 후처리)을 걸어 갑자기 중국어가 튀어나오는 오류를 완벽히 차단했습니다.
  • FastAPI를 활용한 '끊김 없는 연속 대화' 구현: 사용자의 음성과 영상을 서버가 통째로 다 받을 때까지 기다리지 않고, 처리된 조각(Chunk)부터 순차적으로 바로바로 사용자에게 전달(Streaming API)하여 실제 사람과 대화하는 듯한 UX(사용자 경험)를 완성했습니다.

기술 스택

Python FastAPI PyTorch Qwen2-VL-2B (VLM) M2M100 (Translation) Web Speech API (STT/TTS) React

AI 인터뷰 코치

웹캠 딥러닝 기반 실시간 면접 태도 분석 시스템

1. 기획 배경

취업 준비생들이 면접 과정에서 내용적인 측면뿐만 아니라 자신도 모르게 드러나는 시선 불안, 제스처, 표정, 떨리는 목소리 등 비언어적 요소로 인해 불이익을 점검해 볼 실전 대비 환경이 마땅치 않음을 파악, 이를 보조해 줄 1:1 트레이닝 환경을 고안했습니다.

2. 프로젝트 개요

비대면 면접 상황을 시뮬레이션하고, 실시간 웹캠 및 마이크 분석을 통해 답변 내용 및 비언어적인 부분(표정, 시선 처리, 자세, 음향적 특성 등)을 정량화하여 다각도로 분석해 피드백하는 시스템입니다.

3. 담당 역할

기획 및 전체 아키텍처 설계 멀티모달 AI 파이프라인 구축 풀스택(백/프론트엔드) 통합 개발

4. 주요 도전 과제

  • 눈에 보이지 않는 태도를 수치화하기: 면접관이 무의식적으로 느끼는 '시선 불안', '경직된 어깨', '미세한 표정 변화' 같은 모호한 비언어적 요소들을 AI가 어떻게 객관적인 점수로 평가하게 만들 것인지가 가장 큰 고민이었습니다.
  • 눈, 귀, 뇌를 동시에 굴리기 (멀티모달 통합): 영상을 살피는 AI(눈), 목소리의 떨림을 듣는 AI(귀), 답변의 논리를 분석하는 AI(뇌)가 동시에 작동하면서도 서로 충돌하거나 느려지지 않게 하나로 매끄럽게 엮어야 했습니다.
  • 가볍고 빠른 피드백 유지: 여러 개의 AI를 동시에 띄우면 컴퓨터가 쉽게 뻗어버릴 수 있습니다. 일반 웹캠 환경에서도 지연 없이 실시간으로 피드백을 주려면 AI 모델들을 최대한 가볍게(경량화) 유지하는 것이 필수적이었습니다.

5. 성과 및 기여도

  • 미세한 표정과 흔들리는 시선 잡아내기: DeepFace 기술을 응용해 사용자가 당황할 때 짓는 표정을 프레임 단위로 읽어내고, MediaPipe를 통해 면접 내내 동공의 초점이 가운데를 잘 유지하는지, 어깨는 펴고 있는지(자세)를 면밀히 추적하는 로직을 완성했습니다.
  • 목소리의 떨림까지 캐치하는 AI: 단순히 사용자가 '무엇을' 말했는지 텍스트로 바꾸는 것(Whisper)을 넘어, 말할 때의 긴장도나 목소리 떨림 같은 감정 상태까지 분석(Wav2Vec2)하여 면접관의 귀를 대신하는 파이프라인을 구축했습니다.
  • 무겁지 않고 똑똑한 코칭 리포트 (Qwen 0.5B): 면접이 끝난 직후 "여기를 이렇게 고쳐보세요" 하거나 자연스러운 꼬리 질문을 던지는 역할은 아주 작고 가벼운 모델(0.5B)에게 맡겨, 응답 속도를 일반 챗봇 수준으로 빠르게 끌어올렸습니다.
  • 빠릿빠릿한 1:1 비대면 AI 면접 환경: 결과적으로 영상, 음성, 텍스트 분석에 특화된 가장 가벼운 모델들만 쏙쏙 골라 조립함으로써, 값비싼 서버 장비 없이도 부드럽게 동작하는 실전 면접 트레이닝 환경을 구축해 냈습니다.

기술 스택

Python FastAPI Qwen2.5-0.5B-Instruct Whisper-tiny Wav2Vec2 (Emotion Recognition) DeepFace (OpenCV) MediaPipe (FaceMesh, Pose)

DDauction 땅땅옥션

대규모 경매/중고거래 통합 커머스 플랫폼

1. 기획 배경

개인 간 중고 거래 시 가격 측정의 기준이 모호해 판매 상품 등록에 어려움을 겪거나 채팅을 통해 불필요한 가격 조정 과정을 거쳐야 하는 불편함이 다수 관찰되었습니다. 일본의 대형 플랫폼인 야후 옥션, 번개장터, KREAM의 성공 사례에서 영감을 받아 기획했습니다.

2. 프로젝트 개요

실시간 경매와 개인 중고거래를 하나의 플랫폼에서 제공하는 통합형 커머스 플랫폼입니다. 실시간 입찰처리와 안전한 인증 시스템이 핵심입니다.

3. 담당 역할

프론트엔드 DB 설계 백엔드

4. 주요 도전 과제

  • 실시간 동기화 이슈 해결: 10명이 동시에 입찰 버튼을 눌렀을 때, 누구의 입찰가를 먼저 인정할 것인지, 그리고 바뀐 가격을 모든 사람의 화면에 어떻게 지연 없이 보여줄 것인지가 가장 시급한 문제였습니다.
  • 대용량 트래픽 및 데이터 처리 최적화: 경매 마감 직전이나 인기 상품이 올라왔을 때 트래픽이 몰려 서버가 다운되는 것을 막기 위해, 수만 개의 상품과 입찰 데이터를 빠르고 안전하게 처리할 수 있는 데이터베이스 설계가 필요했습니다.
  • 사용자 경험(UX) 극대화: 거래 앱의 생명은 '알림'과 '소통'입니다. 내 입찰가가 밀려났을 때 즉각적인 푸시 알림을 주고, 판매자와 구매자가 1:1로 원활하게 대화할 수 있는 생동감 있는 환경을 구축해야 했습니다.

5. 성과 및 기여도

  • 동시성(Concurrency) 제어 최적화: 여러 명이 동시에 입찰하려 할 때, 데이터가 꼬이지 않도록 은행 출금 시스템처럼 철저하게 순서를 세우는 안전장치(트랜잭션 최적화)를 구현하여 신뢰성 있는 실시간 경매 시스템을 완성했습니다.
  • WebSocket 기반 실시간 통신망 구축: '새로고침' 버튼을 누르지 않아도, 입찰가 갱신, 1:1 채팅 메시지, 낙찰 알림 등이 카카오톡처럼 실시간으로 화면에 동기화되도록 STOMP 통신 인프라를 전면 구축했습니다.
  • 확장성 높은 데이터베이스 아키텍처(ERD) 설계: 상품 정보, 입찰 기록, 회원 정보가 한통속으로 섞여 느려지지 않도록, 대용량 처리에 맞게 데이터를 잘게 쪼개고 체계적으로 분류(데이터베이스 정규화)하여 서비스 덩치가 커져도 견딜 수 있는 뼈대를 세웠습니다.
  • 안전한 통합 인증 체계(Spring Security / JWT): 사용자가 복잡한 가입 절차 없이 네이버, 카카오, 구글 아이디로 클릭 한 번에 안전하게 접속할 수 있는 OAuth2 로그인 시스템을 완성했습니다.

01

조회수 중복 증가 및 데이터 신뢰성 문제 해결

문제 상황

상품 상세 페이지 진입 시 새로고침(F5)이나 프론트엔드의 API 중복 호출로 인해 조회수가 비정상적으로 부풀려지는 현상 발생. 단순 수정(@Modifying) 쿼리 사용으로 인해 트랜잭션 내 세밀한 제어가 어려워 데이터 신뢰성이 저하됨.

해결 방안

① 조회 로그 테이블(ProductViewLog) 및 쿨다운 도입
유저별 조회 이력을 기록하여, 동일 유저가 1시간 이내에 다시 접속할 경우 조회수가 중복으로 집계되지 않도록 제어했습니다.

[조회 로직 플로우]
상세 페이지 진입 → DB에서 해당 유저의 최근 조회 기록 확인
  ↓
기록 없음?       → 조회수 +1, 새로운 로그 생성
1시간 경과?       → 조회수 +1, 기존 로그 갱신
1시간 이내?       → 조회수 유지 (아무 작업 하지 않음) ✅

② API 요청 목적의 명확한 분리
?incrementView=false 파라미터를 도입하여, 화면 렌더링을 위한 단순 데이터 요청과 조회수 카운트 요청을 기술적으로 분리함으로써 중복 호출에 따른 부작용을 원천 차단했습니다.
