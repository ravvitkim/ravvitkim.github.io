Problem Solving

트러블슈팅 경험

01

하드코딩 분기 로직 → ReAct 아키텍처 전환

문제 상황

특정 키워드에만 반응하는 if/else 구조, 버전 정보 부족 시 실패하거나 부정확한 응답 발생

해결 방안

LangGraph 기반 ReAct(Reasoning + Acting) 아키텍처 도입, LLM이 스스로 판단하여 필요한 Tool을 단계적으로 호출

개선 결과

정적 분기 로직에서 자율 추론 기반 에이전트로 진화

02

단순 텍스트 슬라이싱 → 구조적 차이 분석

문제 상황

문서 앞 2000자만 전달하는 단순 슬라이싱으로 후반부 변경사항 누락, SOP/GMP 조항 단위 분석 불가

해결 방안

DB 레벨에서 조항 비교 기능으로 ADDED / DELETED / MODIFIED 별 상태 명확 구분

개선 결과

대용량 문서에서도 누락 없는 비교, 리포트 신뢰도 대폭 향상

03

Neo4j 기반 Impact Analysis 도구 연동

문제 상황

변경 영향 섹션에서 실제 영향을 받는 문서 목록 누락, 추상적 영향 평가에 그침

해결 방안

Neo4j 그래프 DB 기반 get_impact_analysis_tool 신규 추가, 에이전트가 영향 분석 Tool을 반드시 호출하도록 프롬프트 강화

개선 결과

변경 문서를 인용하는 타 문서 목록 자동 추출, 실질적인 감사 대응 리포트 수준 확보

04

조항 파싱 및 청킹 최적화

문제 상황

PDF 변환 시 반복 헤더/푸터가 조항 번호로 오인, RAG 검색 품질 저하

해결 방안

동적 노이즈 제거(Dynamic Noise Cleaning) 적용, 문서 전역에서 반복 패턴 탐지 후 제거

개선 결과

노이즈 청크 95% 이상 제거, RAG 검색 정확도 비약적 향상
